myFlix API

This API is the server-side component of a “movies” web application. The application will provide users with access to information about different  movies, directors, and genres. Users will be able to sign up, update their  personal information, and create a list of their favorite movies.

URL endpoints and request methods

Business Logic URL HTTP Method Request body data format Response body data format
Get a list of all movies /movies GET None A JSON object holding data about all the movies
Get data about a single movie by title to the user /movies/:title GET None A JSON object holding data about a single movie including its description, genre, director, image URL and whether it's featured or not
Get data about a genre (description) by name /movies/genre/:genreName GET None A JSON object containing all data about a single genre including its name (e.g., "Thriller") and description
Get data about a director by name /movies/directors/:directorName GET None A JSON object holding data about a director (bio, birth year, death year) by name
Get a list of all users /users GET None A JSON object holding data about all the users
Get user by username /users/:username GET None If the user doesn't exist, a text message stating that the user with that name doesn't exist, if the user exists a JSON object holding data about the users
Allow new users to register /users POST A JSON object holding data about the new user including their username, and favorite movies. Required: Username, Password, Email { "Username": "Joe", "Password": "pw123", "Email": "Joe@mail.com", "Birth_date": "YYYY-MM-DD" } If the user already exits, a text message indicating that a user with this Username already exists. If the user is new a JSON object holding data of the new user and ID { "Username": "Joe", "Password": "pw123", "Email": "Joe@mail.com", "Birth_date": "2000-12-31T23:00:00.000Z", "FavoriteMovies": [], "_id": "6303ad5d2b46ea59a555ee46", "__v": 0 }
Allow users to update their user info /users/:Username PUT A JSON object containing data about the user. Required: Username, Password, Email { "Username": "Joe", "Password": "pw123", "Email": "Joe@mail.com", "Birth_date": "YYYY-MM-DD" } A JSON object with the updated user information { "Username": "Joe", "Password": "pw123", "Email": "Joe@mail.com", "Birth_date": "2000-12-31T23:00:00.000Z", "FavoriteMovies": [], "_id": "6303ad5d2b46ea59a555ee46", "__v": 0 }
Allow users to add a movie to their list of favorites /users/:Username/:movieId POST None A JSON object with the updated user information { "Username": "Joe", "Password": "pw123", "Email": "Joe@mail.com", "Birth_date": "2000-12-31T23:00:00.000Z", "FavoriteMovies": [], "_id": "6303ad5d2b46ea59a555ee46", "__v": 0 }
Allow users to remove a movie from their list of favorites /users/:Username/:movieId DELETE None A text message indicating whether or not the movie was successfully removed from the user's favorite list
Allow existing users to deregister by User ID /users/:Userid DELETE None A text message indicating that the user was successfully removed of that the user was not found
